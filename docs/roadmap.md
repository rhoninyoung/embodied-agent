# 项目路线图

## 整体节奏

```
Phase 0          Phase 1           Phase 2           Phase 3          Phase 4
能看见  ──────▶  能动手  ──────▶  能制造  ──────▶  能感知精细  ──▶  自举
(1-2周)          (2-4周)           (2-3周)           (4-8周)         (持续)
```

---

## Phase 0: 视觉感知 MVP

**目标**: AI 可以通过摄像头"看见"工作台，并控制观察角度。

**视觉模式**: 默认 **L0（按需快照）**— AI 仅在需要时主动请求拍照，最大限度控制 token 消耗。在此阶段验证基础视觉能力后，后续按需升级到 L1 事件驱动模式。

### 里程碑

- [ ] P0.1 — USB 摄像头接入，AI 可以请求拍照并分析图像（L0 按需模式）
- [ ] P0.2 — 舵机云台组装，AI 可以控制摄像头的俯仰/旋转
- [ ] P0.3 — 本地中间件基础框架：WebSocket 服务 + 设备管理
- [ ] P0.4 — 视觉交互级别管理：支持 L0/L1 动态切换，token 消耗日志

### 交付物

- `src/middleware/server.py` — WebSocket 服务端
- `src/vision/camera.py` — 摄像头驱动
- `src/vision/ptz_control.py` — 云台控制
- `src/vision/vision_level.py` — 视觉交互级别管理器
- Arduino 固件：接收串口指令驱动云台舵机

### 验收标准

> AI 发出"向左转30度，拍一张照"的指令（L0 按需模式），能收到正确角度的照片并描述画面内容。Token 消耗有日志记录。

## Phase 1: 机械臂控制

**目标**: AI 能控制机械臂执行基础的抓取-放置操作。

**视觉模式**: **L1（事件驱动）**— 机械臂每步动作完成后，中间件自动拍照上报确认结果。

### 里程碑

- [ ] P1.1 — 机械臂串口通信建立，可控制各关节角度
- [ ] P1.2 — 正/逆运动学实现，支持笛卡尔空间定位
- [ ] P1.3 — 夹爪控制：开、合、力度调节
- [ ] P1.4 — 视觉引导抓取：看到物体 → 计算位置 → 抓取
- [ ] P1.5 — 安全约束引擎上线：工作空间围栏、力限制、急停

### 交付物

- `src/control/arm_driver.py` — 机械臂串口驱动
- `src/control/kinematics.py` — 运动学求解
- `src/control/gripper.py` — 夹爪控制
- `src/middleware/safety.py` — 安全约束引擎
- 手眼标定数据

### 验收标准

> AI 看到桌面上一个积木块，自主规划路径，抓起来放到指定位置，误差 < 5mm。

---

## Phase 2: 3D 打印集成

**目标**: AI 能完成从设计到打印的全自动流程。

**视觉模式**: **L2（低速轮询）**— 打印过程中每 5\~10 秒拍照监控，异常时自动升到 L1 上报。

### 里程碑

- [ ] P2.1 — 拓竹 P2S 局域网连接，MQTT 通信建立
- [ ] P2.2 — OpenSCAD 脚本生成 → STL 编译管线
- [ ] P2.3 — PrusaSlicer CLI 集成，自动切片
- [ ] P2.4 — 打印任务提交与状态监控
- [ ] P2.5 — 端到端验证：AI 设计一个手机支架并打印出来

### 交付物

- `src/printer/bambu_client.py` — 拓竹 MQTT 客户端
- `src/printer/model_gen.py` — OpenSCAD 建模管线
- `src/printer/slicer.py` — 切片引擎封装

### 验收标准

> 用户说"给我打印一个直径5cm的齿轮"，AI 自动建模、切片、发送打印任务，并监控到打印完成。

---

## Phase 3: 传感器增强与精密操作

**目标**: 增加力反馈和深度感知，机械臂可以完成更精密的操作。

**视觉模式**: **L3（关键帧上报）**— 本地 30fps 采集，中间件智能选取关键帧上报 AI，兼顾实时性与成本。

### 里程碑

- [ ] P3.1 — 夹爪力传感器集成，实现力控抓取
- [ ] P3.2 — 深度相机接入，获取 3D 点云
- [ ] P3.3 — 点云 → 工件位姿估计
- [ ] P3.4 — 精密插装操作（如将销钉插入孔位）
- [ ] P3.5 — 焊接模块验证：AI 控制烙铁完成一个焊点

### 交付物

- 力控夹爪硬件 + 驱动
- 深度相机驱动与点云处理管线
- 焊接末端执行器

### 验收标准

> AI 从料盘中拾取一个电阻，识别其阻值，精确放置到 PCB 指定位置，并完成焊接。

## Phase 4: 硬件自举

**目标**: AI 利用现有工具制造更好的工具。

**核心策略**: 混合制造 — 3D 打印做连接件/外壳，购买金属件（谐波减速器、轴承、碳纤管）做骨架和传动。不追求"全打印"，追求"超越 myCobot 280 的精度"。

**精度目标**: 从 myCobot 280 的 ±0.5mm → 自制臂 ±0.2\~0.3mm（可行性评估 50\~60%）

### 自举路径（精度递进）

```
myCobot 280 (±0.5mm)
  │
  ├── 第一步: 用 myCobot + 人工辅助装配 "自制臂 v1"
  │           目标: 匹配 myCobot (±0.5mm)，验证机械设计
  │
  ├── 第二步: 软件补偿 + 视觉校准，提升 v1 有效精度
  │           目标: ±0.3mm (软件补偿增益)
  │
  ├── 第三步: 用 v1 臂 + AI 迭代设计 → 装配 "自制臂 v2"
  │           目标: ±0.2mm (设计优化 + 更好装配)
  │
  └── 持续迭代...
```

### 关键设计原则

- **可调节关节**: 预留偏心凸轮/调节螺丝位置，装配后可微调对齐
- **软件补偿**: 运动学标定 + 视觉闭环，消除机械误差
- **混合结构**: 碳纤管/铝型材做骨架（刚性），3D 打印做连接件（快速迭代）
- **精度瓶颈突破**: 谐波减速器消除回差 → 这是超越 myCobot 最大的单一增益

### 里程碑

- [ ] P4.1 — AI 分析当前 myCobot 280 的精度瓶颈（回差测量、刚性测试）
- [ ] P4.2 — AI 设计混合结构机械臂 v1（OpenSCAD 连接件 + 购买金属骨架）
- [ ] P4.3 — 3D 打印连接件，人工辅助装配关键传动件（谐波减速器、轴承）
- [ ] P4.4 — 运动学标定 + 视觉校准 + 软件补偿
- [ ] P4.5 — 精度对比测试：自制臂 vs myCobot 280
- [ ] P4.6 — 迭代改进：用 v1 臂辅助装配 v2 臂

### 验收标准

> AI 独立完成一次机械臂的升级迭代：分析 myCobot 精度瓶颈 → 设计改进版（混合结构）→ 打印连接件 → 装配 → 标定校准 → 精度测试超越 myCobot 280。

### 自举可行性与限制（诚实评估）

| 目标 | 把握 | 关键依赖 |
|------|------|---------|
| 匹配 myCobot 280（±0.5mm） | 85\~90% | 合理结构设计 + 闭环步进电机 |
| 超越至 ±0.2\~0.3mm | 50\~60% | 谐波减速器 + 交叉滚子轴承 + 软件补偿 |
| 达到 ±0.1mm | 20\~30% | 以上全部 + 多轮迭代 + 金属骨架 |
| 接近工业级 ±0.05mm | <10% | 需 CNC 加工，超出 FDM 打印能力边界 |

**底线**: FDM 打印件精度极限约 ±0.1\~0.2mm，材料刚性远不如金属。完全 3D 打印的手臂无法超越金属结构。混合制造是必经之路。

## 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 机械臂精度不足 | P1, P3 进度受阻 | 先用成品臂验证软件链路，再自制 |
| 拓竹 MQTT 协议变更 | P2 打印控制失效 | 留 FTP 上传备选方案 |
| 实时性不满足 | 精密操作失败 | 中间件做更多本地决策，减少 AI 往返 |
| 摄像头标定漂移 | 视觉定位不准 | 定期自动校准程序 |
| 安全事故 | 硬件损坏/人员伤害 | 安全引擎必须在 Phase 1 前完成 |
| Token 消耗过快 | 订阅额度耗尽、开发中断 | 默认 L0 按需模式，逐步升级；token 消耗日志监控 |
